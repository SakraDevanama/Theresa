## 1. 修复 UnseenFuture 递归崩溃

### 问题
`UnseenFuture`（明日渺远不及）被 `DustManager.DustIt()` 打出时，会创建副本并通过 `CardCmd.AutoPlay` 自动打出。副本的 `OnPlay` 会再次调用 `ProcessDustEffect()` → `DustManager.DustIt()` → 再打出另一张副本，形成**无限递归**。

最终抛出：
```
System.InvalidOperationException: Card must be in a combat pile for it to be removed
```

### 修复
在 `UnseenFuture` 中使用 `AsyncLocal<bool>` 标志防止递归：

```csharp
private static readonly System.Threading.AsyncLocal<bool> IsProcessingDustEffect = new();

private async Task ProcessDustEffect()
{
    if (IsProcessingDustEffect.Value) return; // 防止递归
    IsProcessingDustEffect.Value = true;
    try { /* ... */ }
    finally { IsProcessingDustEffect.Value = false; }
}
```

**文件**: `TheresaCode/Cards/UnseenFuture.cs`

## 7. 修复 UnknownRelic 升级替换问题

### 问题
`UnknownRelic.GetUpgradeReplacement()` 使用 `ModelDb.Relic<KnownRelic>()` 获取**不可变原型**，然后调用 `SetInitialActionCount` 修改它。但 `TouchOfOrobas.AfterObtained()` 会调用 `.ToMutable()` 创建副本，导致修改丢失。

结果：
- `UnknownRelic` 没有被正确替换为 `KnownRelic`
- 或者替换后 `KnownRelic` 的 Action 计数没有正确转移

### 修复
直接返回 `.ToMutable()` 后的副本：

```csharp
public override RelicModel? GetUpgradeReplacement()
{
    // 创建 KnownRelic 的可变副本并转移计数
    var knownRelic = ModelDb.Relic<KnownRelic>().ToMutable();
    if (knownRelic is KnownRelic kr)
    {
        kr.SetInitialActionCount(ActionCount);
    }
    return knownRelic;
}
```

**文件**: `TheresaCode/Relics/UnknownRelic.cs`

## 技术要点总结

### `GetUpgradeReplacement()` 调用链
1. 先古之民事件获得 `TouchOfOrobas`
2. `TouchOfOrobas.AfterObtained()` 调用 `GetUpgradedStarterRelic()`
3. `BaseLib.StarterUpgradePatches` 拦截，调用 `CustomRelicModel.GetUpgradeReplacement()`
4. 返回非 null 时，使用 `RelicCmd.Replace` 替换原有 starter 遗物

### 关键注意事项
- `ModelDb.Relic<T>()` 返回的是**不可变原型**，修改后如果调用方再 `.ToMutable()`，修改会丢失
- `AsyncLocal<T>` 可以跨异步调用链传递状态，适合防止递归
- `HoverTipFactory.FromPower<T>()` 和 `HoverTipFactory.FromKeyword()` 是添加卡牌提示的标准方式
